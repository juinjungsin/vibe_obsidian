바이브 옵시디언
나만의 PKM, 허들을 넘다

생성형 AI가 옵시디언 커뮤니티 플러그인에 연결되면서, 노동 집약적이던 PKM 구축이 "일단 해보는" 영역으로 바뀌었다. 복잡한 개념과 명령어는 잠시 미뤄두자. 내가 가진 자료로 그래프 뷰부터 만들어본다.

강의 시간 4H · 실습형 대상 PKM 입문~중급 도구 Obsidian · Claude · Codex 작성일 2026-05-18
00 / OVERVIEW

왜 지금, 바이브 옵시디언인가

PKM(개인지식관리체계)은 전문가들의 전유물이었다. 다양한 도구들이 존재했고, 사용자 분포는 대체로 다음과 같다.

PKM 사용자 분포

허들을 넘지 못한 70% · 넘었지만 유지 못 한 20% · 유지하지만 갈 길 잃은 5% · 프론티어 5%

강의의 목표 지점

프론티어 5%는 동경의 대상. 이 강의가 다루는 구간은 "허들 넘기"와 "유지하기".

강의 포지셔닝

옵시디언은 자유도가 높고 좋은 기능을 가지고 있지만, 의미 있는 수준에 도달하기까지 작업자의 노동 집약적 요소가 많았다. 그것이 실제로 허들로 작용했다. 옵시디언의 커뮤니티 플러그인에 생성형 AI가 연결되면서, 기존과 다른 접근으로 옵시디언을 사용할 계기가 마련되었다. 이른바 바이브 옵시디언.

01 / CONCEPT

무엇이 달라지는가

근거: Karpathy LLM Wiki Gist (2026-04-04)

핵심은 합성 시점의 이동. RAG/NotebookLM은 질문이 들어올 때마다 원본을 다시 합성하지만, LLM Wiki는 ingest 시점에 합성을 끝낸다. 답변은 다시 위키로 저장된다 — Karpathy 표현으로 compounding.

시간 경과에 따른 가치 곡선

RAG는 6개월 뒤에도 1일차와 동일 출발선. LLM Wiki는 페이지·교차 참조가 누적될수록 가속.

R
RAG / NotebookLM
원본 → 질문 → 청크 검색 + 합성 (매번 처음부터) → 답변 (휘발). 누적되는 것이 없음.
W
LLM Wiki (바이브 옵시디언)
원본 → ingest 시점 합성 → 영속 위키 → 질문 시 위키 읽음 → 답변 + 위키 추가. 사용할수록 가치가 커지는 자산.

02 / CURRICULUM

강의 구성 · 7개 모듈

총 4시간 · 실습 비중 60%
1
일단 봅시다
25분
  • 강사의 옵시디언으로 만든 결과물 시연 — vault, 그래프 뷰, 자동 생성 인덱스
  • 결과물로 무엇을 할 수 있는가 — 제안서 작성, 강의안 합성, 리포트 출력
  • 지식의 복리 효과 — 질문 → 답변 → 위키 누적의 양방향 흐름
2
나도 해봅시다
15분
  • 준비물 1 — Claude / Codex / Copilot 유료 계정 또는 API 사용료
  • 준비물 2 — 노트북 (Claude 사용 시 CLI 설치 환경)
  • 자료 준비 — PDF, TXT, HTML, JSON, MD 등 정제된 소스
정제된 소스를 넣지 않으면 오류가 위키에 baked-in되어 미래 답변까지 오염된다. 조건이 가벼운 게 아니라 시스템 성패의 핵심 변수.— 강의 자료 3.3 개선 흐름
3
따라해봅시다 (설치)
35분
  • 옵시디언 설치 — 1 PC, 1 vault 원칙
  • 화면 구성 — Editor, Graph View, File Explorer, Side Panel
  • 커뮤니티 플러그인 설치 — Claudian (Claude Code/Codex를 사이드바에 embed)
  • Claude Code 사용자 — CLI 설치 과정 추가
  • Codex 사용자 — 별도 설치 없음
  • 유료 계정 인증 과정 — 토큰/세션 설정
4
따라해봅시다 (raw · wiki · 그래프 뷰)
45분
  • 폴더 지정 — 로컬 기본, 하드 용량·백업·Cloud Drive 동기화 고려
  • 자료가 준비되었는가 — raw/ 폴더 입력 형식 점검
  • 나는 무엇을 하고 싶은가 — 지식 욕구의 방향성 정의 (스키마 CLAUDE.md에 반영)

따라해봅시다 (자동화)
40분
  • 스케줄 수집 — 매일 / 12시간 간격으로 A 주제 조사하여 특정 폴더 저장
  • 폴더 감시 — 파일 유입 시 옵시디언이 자동 확인 후 wiki로 등록, 그래프 뷰 반영
  • 3가지 운영(Operations) — Ingest · Query · Lint 주기 설계
6
문제가 생겼어 (트러블슈팅)
30분
  • 커뮤니티 플러그인이 동작하지 않아요 — CLI 미설치 / 포트 충돌(27124) / 인증 만료
  • 자동화가 멈추었어요 — 토큰 한도 / 폴더 권한 / 스케줄러 비활성
  • 위키가 이상해요 — Referential Integrity 깨짐, Lint 미실행, 모순 페이지
7

  • 외부에서 확인하고 싶어 — Slack 연동, Sync, Obsidian Publish
  • 다음 단계 — 부서/팀 vault 분리, 도메인별 그래프
